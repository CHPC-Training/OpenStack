# Deploying Applications

## 3.1. Monitoring with Grafana and Prometheus

- Login to the VM that you want the Grafana and Prometheus services to be hosted on.
- Add the relevant repository to the package manager
  ```bash
  $ sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
  ```
- Install the relevant packages
  ```bash
  $ sudo dnf install docker-ce docker-ce-cli containerd.io docker-compose-plugin
  ```
- Start the docker service
  ```bash
  $ sudo systemctl start docker 
  ```
- Enable the service to run whenever the system reboots
  ```bash
  $ sudo systemctl enable docker 
  ```
- Check that the docker command is available
  ```bash
  $ sudo docker ps
  ```
- Create a directory to store relevant files
  ```bash
  $ sudo mkdir /opt/monitoring_stack 
  ```
- Create the file `/opt/monitoring_stack/docker-compose.yml` containing
  ```bash
  services:
    node-exporter:
      image: prom/node-exporter
      ports:
        - "9100:9100"
      restart: always
      networks:
        - monitoring-network

    prometheus:
      image: prom/prometheus
      ports:
        - "9090:9090"
      restart: always
      volumes:
        - /opt/monitoring_stack/prometheus.yml:/etc/prometheus/prometheus.yml
      networks:
        - monitoring-network

    grafana:
      image: grafana/grafana
      ports:
        - "3000:3000"
      restart: always
      environment:
        GF_SECURITY_ADMIN_PASSWORD: admin
      volumes:
        - /opt/monitoring_stack/prometheus-datasource.yaml:/etc/grafana/provisioning/datasources/prometheus-datasource.yaml
      networks:
        - monitoring-network

  networks:
    monitoring-network:
      driver: bridge
  ```
- Create another file `/opt/monitoring_stack/prometheus-datasource.yaml` containing
  ```bash
  apiVersion: 1

  datasources:
    - name: Prometheus
      type: prometheus
      access: proxy
      url: http://prometheus:9090
  ```
- Create the final dependent file `/opt/monitoring_stack/prometheus.yml` containing
  ```bash
  global:
  scrape_interval: 15s

  scrape_configs:
    - job_name: 'node-exporter'
      static_configs:
        - targets: ["<private_ip_address_of_grafana_server_host>:9100"]
  ```
- Deploy the containerised monitoring services by executing the following command within the `/opt/monitoring_stack/` path
  ```bash
  $ sudo docker compose up -d"
  ```
- Access your Grafana web server externally by
  - Creating another security group for port 3000
  - Tunnelling in to your VM that's hosting the Grafana server
  - Pointing your browser to `https://localhost:<tunnelled-port>`

- Import the Node Exporter dashboard via the ID `1860` to visualise the metrics being collected for the VM hosting the
  Grafana server.
- In order to visualise additional hosts that are also running the Node Exporter service
  - Modify the `/opt/monitoring_stack/prometheus.yml` file on the VM hosting the Grafana server, specifically the
    targets section to look something like: `- targets: ["<private_ip_address_of_grafana_server_host>:9100",
    "<private_ip_address_of_additional_host1>:9100", "<private_ip_address_of_additional_host2>:9100"]`
  - Restart the Prometheus service
    ```bash
    $ sudo docker compose restart prometheus
    ```
  - Visit the Node Exporter dashboard to see the additional nodes available from the dropdown.
